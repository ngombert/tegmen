# Story 3.1: Observabilité et Traçabilité (Logging Structuré & Correlation ID)

Status: done

## Story

As a Developer,
I want to implement structured JSON logging and propagate correlation IDs,
so that I can easily debug distributed A2A requests between Maestro and Acadomie.

## Acceptance Criteria

1. **Given** an A2A request containing a `correlation_id` in the payload **When** the Acadomie agent processes the request **Then** it logs every significant event (request received, tool call, response sent) in JSON format.
2. **Given** a log entry generated by the agent **When** the log is formatted **Then** it includes the unique `correlation_id` (already partially implemented via `@with_context`).
3. **Given** a log message containing potential PII (Personally Identifiable Information) **When** the log is formatted **Then** the PII (like `family_id`, `student_id`, or names) are automatically redacted from the logs.
4. **Given** the logging setup **When** tests are run **Then** the correlation ID propagation and PII redaction are fully covered by unit tests.

## Tasks / Subtasks

- [ ] **Task 1 — Améliorer le `JSONFormatter` pour la rédaction des PII** (AC: #3)
  - [ ] 1.1 Modifier `app/logger.py` pour ajouter une logique de "Redaction" (masquage) dans le `JSONFormatter`.
  - [ ] 1.2 Identifier et remplacer les patterns sensibles (ex: `family_id`, `student_id` ou emails potentiels) par des étoiles `***` ou un hash.
- [ ] **Task 2 — Vérifier les logs existants** (AC: #1)
  - [ ] 2.1 S'assurer que les services (`a2a.py`, `llm_service.py`, `grades_service.py`, `calendar_service.py`, `organization_service.py`) logguent correctement les requêtes entrantes et erreurs sans fuiter de PII (par exemple en évitant de logger la structure de la requête entière si elle contient des infos personnelles).
- [ ] **Task 3 — Tests Unitaires** (AC: #4)
  - [ ] 3.1 Créer `tests/test_agent_acadomie/test_logger.py` pour valider le `JSONFormatter`.
  - [ ] 3.2 Vérifier que `correlation_id` est injecté s'il existe dans le contexte.
  - [ ] 3.3 Vérifier que le mécanisme de rédaction de PII masque bien les données sensibles dans le champ `message`.

## Dev Notes

- La propagation basique du contexte est déjà en place grâce au fichier `context.py` et au décorateur `@with_context`. L'effort ici porte principalement sur la sécurisation (PII) et la formalisation par les tests.
- Pour le PII Redactor, un filtre simple par expression régulière sur les clés sensibles (ex: regex cherchant `fam-[a-zA-Z0-9]+` ou un masquage ciblé sur les logs d'erreurs) suffit pour ce MVP.
